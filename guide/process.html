流程规范 ​

为了让协会资料长期可维护，建议按固定流程整理和发布。

上传流程 ​

  1. 按主题把资料放入 uploads/ 对应文件夹。
  2. 运行 npm run sync:uploads。
  3. 本地打开资料库，确认标题、预览和下载链接正常。
  4. 运行 npm run docs:deploy 发布到线上。

文件命名 ​

  • 文件名尽量具体，例如 STM32-GPIO-入门.md。
  • 同一系列资料建议放入同一个子目录。
  • 不要使用只对个人有意义的名字，例如 新建文档1.docx。

内容质量 ​

  • 每份资料尽量写清楚适用对象、前置知识和实验环境。
  • 项目复盘建议包含目标、硬件清单、软件环境、关键问题和改进方向。
  • 过期资料可以保留，但应在标题或正文中注明版本。
